# Design Tokens

Design tokens are the visual design atoms of the DeDevs UI Design System. They define the core values for colors, typography, spacing, and other visual properties that ensure consistency across all components and applications.

## Color System

The color system is built on CSS custom properties that support both light and dark themes automatically.

### CSS Variables

All colors are defined as HSL values in CSS custom properties:

```css
:root {
  --background: 0 0% 100%;
  --foreground: 222.2 84% 4.9%;
  --primary: 221.2 83.2% 53.3%;
  --primary-foreground: 210 40% 98%;
  --secondary: 210 40% 96%;
  --secondary-foreground: 222.2 84% 4.9%;
  --muted: 210 40% 96%;
  --muted-foreground: 215.4 16.3% 46.9%;
  --accent: 210 40% 96%;
  --accent-foreground: 222.2 84% 4.9%;
  --destructive: 0 84.2% 60.2%;
  --destructive-foreground: 210 40% 98%;
  --border: 214.3 31.8% 91.4%;
  --input: 214.3 31.8% 91.4%;
  --ring: 221.2 83.2% 53.3%;
}
```

## Usage Guidelines

### Consistency
- Use design tokens consistently across all components
- Avoid hardcoded values in favor of token references
- Maintain the established visual hierarchy

### Customization
- Override CSS custom properties to customize themes
- Extend the Tailwind configuration for additional tokens
- Use semantic color names rather than specific hues

### Accessibility
- Ensure sufficient color contrast ratios (WCAG AA: 4.5:1)
- Test color combinations in both light and dark modes
- Provide alternative indicators beyond color alone

The design token system ensures visual consistency while providing the flexibility needed for domain-specific customizations across AI interfaces, blockchain applications, and developer portfolios.
